# Contributing

## Ручное тестирование

Расширение не имеет автотестов (логика завязана на живой ответ YouTube
`/player`), поэтому перед PR прогони чек-лист вручную, через **Load unpacked**
(см. [README](README.md#установка)):

- [ ] Жёсткая загрузка видео с авто-дубляжом → через ~1–2 c сам включается
      оригинал.
- [ ] SPA-переход на другое дублированное видео → сразу оригинал, без
      дополнительной перезагрузки.
- [ ] Кнопка **ORIG/DUB** переключает и возвращает дубляж; попап показывает
      то же состояние после перезагрузки.
- [ ] Режим «Не вмешиваться» (`off`) = чистый YouTube, расширение не
      вмешивается в ответ `/player`.
- [ ] Обычное (не дублированное) видео не ломается и не перезагружается
      зря.
- [ ] При `DEBUG = false` (значение по умолчанию) нет ошибок в консоли и нет
      `window.__undubDbg`.

## Если меняешь логику в `inject.js`

- Включи `var DEBUG = true;` — появится `window.__undubDbg` со счётчиками по
  каждому хуку/ветке, это сильно ускоряет диагностику.
- См. [`docs/HOW_IT_WORKS.md`](docs/HOW_IT_WORKS.md) — там описан формат
  ответа `/player`, на который завязана вся логика, и оба пути перехвата
  (XHR/SPA и жёсткая загрузка).
- Перед коммитом верни `DEBUG = false`.

## PR

- Один PR — одно изменение. Опиши, что проверял из чек-листа выше.
- Не меняй версию в `manifest.json` в том же PR, где меняется логика —
  версию поднимает мейнтейнер при релизе.
